## Engineering invariants to preserve

These have been load-bearing across the campaign — don't revert them:

- **WidgetRefresher** uses `MutableSharedFlow + Flow.debounce(400ms) + DROP_OLDEST`. Per-target accumulator (`pendingTargets: AtomicReference<Set<WidgetTarget>>`) drives `refreshMemo` / `refreshToday` / `refreshAll`. Save paths use `refreshAllNow` (inline); sync workers + settings use debounced `refreshAll`.
- **PullWorker / PushWorker** track `roomChanged: Boolean` and only fire `WidgetRefresher` when at least one row mutated.
- **`ServiceLocator.init`** is `synchronized(initLock)` with a re-checked guard inside; concurrent first-call collapses into one builder pass. `applicationContext` is the only Context the singletons hold (StaticFieldLeak suppressed deliberately).
- **`FrontMatterCodec.applyPin/strip/parse`** is the only pin parser. The legacy `MemoRepository.readPinnedFromFrontMatter` is gone. Strict-bool semantic — quoted-value pinned is treated as user YAML.
- **`PathLocker`** is ref-counted (acquire bumps refcount, release drops + GCs at zero) — paired with `withLock` consumers; never naked.
- **`PullBudget`** is `AtomicInteger` + CAS loop. `tightenFromHeader()` ready when a future PR plumbs `X-RateLimit-Remaining` through the API call sites.
- **Composite indices** match the entity declarations one-to-one. Adding/removing an index column means bumping schema + writing a migration.
- **`Validators on companion object`** (`SettingsViewModel.validateGithubPat` / `validateProviderUrl` / `validateModelName` / `validateApiKey` and `AiClient.isProviderUrlAcceptable`). Loopback http is allowed only for `localhost / 127.0.0.1 / [::1]`.
- **`SharingStarted.WhileSubscribed(5_000L)`** on the three list VMs (NoteList / Tag / Calendar). Don't switch back to Eagerly.
- **`reminderMinutesBefore`** persists via `X-MEMO-REMINDER-MINUTES` X-property. PullWorker prefers the decoded value but falls back to the local row for legacy events without the X-property (backwards-compat).
- **`MigrationDefaultsTest`** is a tripwire. Adding a new migration with a `DEFAULT` clause means adding a new expectation entry in `EXPECTED_DEFAULT_LINES`.
- **AndroidManifest backup**: `dataExtractionRules` (Android 12+) + `fullBackupContent` (pre-12) both excluding root/database/sharedpref/external. Don't switch back to bare `allowBackup="false"`.
- **Reduce-motion**: AppNav reads `Settings.Global.ANIMATOR_DURATION_SCALE` and collapses tab fade to 0ms when 0. Don't hardcode 150ms back.
