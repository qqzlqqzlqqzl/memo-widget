## 这是什么

一个 Android 备忘 + 日程 app，写的内容**全部存在你自己的 GitHub 仓库**里。

- **你就是后端**。开发者看不到你的内容
- **断网能用**。没网先存本地，来网自动传
- **换手机无痛**。新机装上、填回 token，历史全部回来
- **文件透明**。GitHub 网页直接打开就能看

## 首次使用 3 步

### 第 1 步：准备 GitHub 账号和一个仓库

1. 如果还没 GitHub 账号，去 [github.com](https://github.com) 注册
2. 新建一个仓库（可以是**私有**的，推荐叫 `memos` 或 `notes`）。空仓库也行

### 第 2 步：生成一把 Token

1. 去 [https://github.com/settings/tokens/new](https://github.com/settings/tokens/new?scopes=repo&description=memo-widget)
2. **Note**（描述）随便填，比如 `memo-widget`
3. **Expiration**（过期时间）根据需要选，建议 90 天或更长
4. **Select scopes**（勾权限）：**勾上 `repo` 这一整组**（包含 `repo`, `public_repo` 等）
5. 拉到底点 **Generate token**
6. **立刻复制**出现的 `ghp_...` 字符串 — 关页面就看不到了

⚠️ 这个 token 等于仓库密码。**只存你自己手机上**，app 用加密方式存，任何其它地方都不传。

### 第 3 步：装 APK 并填配置

1. 去 [Releases 页](https://github.com/qqzlqqzlqqzl/memo-widget/releases) 下**最新** `app-debug.apk`
2. 手机系统设置 → 应用 → 特殊权限 → 安装未知应用 → 给**你用的浏览器**打勾
3. 点 APK 装
4. 打开 app → 点底部 **设置** tab → 填：
   - **GitHub PAT**：粘贴刚才的 token
   - **Owner**：你的 GitHub 用户名
   - **Repo**：你建的仓库名（如 `memos`）
   - **Branch**：默认 `main` 就行
5. 点蓝色的 **保存** 按钮
6. 顶部状态条变蓝：**当前配置已就绪 ✓**

装好了。

## 日常怎么用

### 写一条备忘

**方法 A**：桌面 2×2 小部件点 **+** → 打字 → **保存**

**方法 B**：打开 app → 底部 **笔记** tab → 右下角 **+ 写一条** → 打字 → **保存**

按了保存就自动提交到 GitHub。一般 1 秒内完成，断网先存本地。

### 看过去写的

底部 **笔记** tab → 倒序列出所有天的备忘。上方搜索框可以全文搜。

### 加日程

1. 底部 **日历** tab
2. 点某一天（默认今天）
3. 右下角 **+ 加日程**
4. 填标题 + 开始时间 + 结束时间
5. 可选：
   - 重复：**每周** / **每月**（或不重复）
   - 提醒：**5 分钟前** / **15 分钟前** / **1 小时前**（或不提醒）
6. **保存**

### 改 / 删日程

日历 → 点那一天 → 点列表里的日程 → 同一个对话框，左下角有 🗑️ 删除按钮

## 桌面小部件

长按桌面空白处 → 添加小部件 → 找到 **备忘** app，有两个部件可选：

| 部件 | 尺寸 | 用处 |
|---|---|---|
| **备忘**（2×2） | 小方块 | 点 + 快速写一条，列表显示今天最近 3 条 |
| **今日**（4×2） | 宽条 | 今天的日程 + 最新备忘一览，点进 app |

## 事件提醒

### 准备工作

**第一次打开 app 会弹通知权限请求**，一定点"允许"。没允许的话提醒不会响。

**系统设置**里还要看一眼：
- 设置 → 通知 → 找到本 app → "日程提醒"通道打开
- 设置 → 应用 → 本 app → 电池优化 → 选"不优化"
- **国产 ROM**（小米、华为、OPPO、vivo 等）：进手机的**自启动管理**，把本 app 加入白名单，**否则手机重启后所有提醒失效**

### 怎么生效

- 加日程时选好"提醒"（5/15/60 分钟前任选）
- 到点手机弹通知，点通知进 app
- 循环日程（每周/每月）会自动排下一次

### 隐私

锁屏只显示"日程提醒"四个字。解锁之后才看到具体内容。

## 多设备同步

想在手机 A + 手机 B + 平板三个地方同时用？**完全可以**。

1. 每台设备都装同一个 APK
2. 都在设置里填**同一个 GitHub token + 同一个 owner/repo**
3. 保存后，所有设备的数据自动从 GitHub 拉下来，以后互相同步

**但**：
- 每台设备的**提醒设置独立**。设备 A 设了"15 分钟前"，设备 B 不会自动有。每台设备单独设
- 两台设备同时改同一天的备忘，极小概率冲突，app 会标"待同步"，联网重试多半能自己解

## 常见问题

### 1. 顶部出现红色"同步失败"条

点"知道了"暂时关掉。可能原因：

- **token 过期** → 回设置页重新生成并填新 token
- **token 权限不对** → 生成时漏勾 `repo` 了，重新来一次
- **网络问题** → 等一下联网会自动重试，不用管
- **仓库不存在** → 去 GitHub 建对应仓库

### 2. 备忘保存了但 GitHub 上看不到

正常情况 1 秒内就上去。如果：
- **没网** → 本地存好了，日期旁有"待同步"标记，来网自动传
- **超过 1 分钟还没上去** → 顶部应该有红条，按上面处理

### 3. 提醒没响

按这个顺序查：
1. 通知权限给了吗？设置 → 应用 → 本 app → 通知 → 打开
2. "日程提醒"这个通道单独没被关？
3. 电池优化有没有豁免？系统会"省电"杀后台
4. 国产 ROM 的**自启动白名单**加了吗？这是最常见的原因
5. Android 12+ 的**闹钟和提醒**权限有没有给？设置里单独一项

### 4. 历史笔记只看到最近的

app 默认只拉近 14 天的增量。**首次安装或清数据时**会自动把仓库里**所有**笔记拉回来，所以你要么：
- 清 app 数据触发全量重拉
- 或者直接去 GitHub 看历史（你的数据透明）

### 5. 换手机怎么迁

1. 新手机装 APK
2. 设置里填同一个 token + owner + repo
3. 打开笔记 tab，第一次会拉全量（10 秒-1 分钟，看你仓库多大）
4. 完事。没有"从旧手机导出"这种步骤

### 6. 想彻底删除 app

1. 卸载 app（本机数据全清）
2. 如果也想删云端数据：去 GitHub 把仓库删了，或者就把 token 撤销（`Settings → Developer settings → Personal access tokens`）

## 你的数据长什么样

打开你的 GitHub 仓库，会看到这样的文件结构：

```
你的仓库/
├── 2026-04-21.md          # 4 月 21 日的备忘（所有条目合在一个文件）
├── 2026-04-22.md
├── 2026-04-23.md
└── events/
    ├── 7f3c-4a2d.ics      # 一个日程一个文件（标准 iCalendar 格式）
    └── 8b21-9c5e.ics
```

**每天的备忘文件**打开来是这样的：

```markdown
# 2026-04-21

## 14:30
今天学了 Glance widget。

## 15:12
- 买菜
- 跑步 30min

## 18:05
晚餐：凉面
```

**日程文件**是标准 iCalendar 格式，可以被 Google 日历、苹果日历等**直接订阅**（订阅方式看各日历 app 的"添加外部日历"）。
